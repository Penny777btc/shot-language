# 镜头语言与画面构图（判断库）

来源：科拉·巴尼克 & 格奥尔格·巴尼克《摄影构图与图像语言》。
本 skill 把静态构图原理转译为**本管线的数字镜头语言**——录屏 punch-in、出镜全景/特写切换、未来的实拍 B-roll。
深度原理目录见 [reference.md](reference.md)；本文件只放**何时用、怎么判断、怎么验收**。

## 核心前提（先记住这一条）

观众在 **0.x 秒、还没看清画面之前**，潜意识已经对画面做了一次"有趣/无聊"的过滤。
画面里**必须有一个最强的"抢眼点"**让视线第一时间落下；**意义单元数量越少越好**（少即是多）。
"设计感"不是加东西，而是**建立主次、把视线引到对的地方、删掉干扰**。

抢眼点的来源（按强度）：人脸/眼睛 > 信号色（红、黄、霓虹）> 高亮/最亮区 > 清晰区（背景虚化时）> 强对比的孤立点 > 圆形。
→ 录屏里，光标、活动输入框、高亮按钮、punch-in 后的清晰区就是抢眼点。**一屏同时只该有一个。**

## 何时调用 / 接入点

| 管线阶段 | 本 skill 提供的判断 |
|---|---|
| 阶段 2 选片（G2） | 哪条 take 的画面"主次清楚、能一眼读懂"——主要元素是否突出、有没有主题重叠/背景混乱 |
| 阶段 5b 运镜 punch-in（G5b） | punch-in 的**本质 = 制造抢眼点 + 降复杂度**。拉近时机、目标区域要落在视觉重心；用景别变换（全景↔特写）制造节奏 |
| 阶段 5b 出镜数字机位 | 全景/特写切换依据景别语言；微仰显自信、平视显亲和、俯视显弱势（见 reference 视角节） |
| 阶段 4 调色（/grade） | 色彩构图：信号色作抢眼点、冷暖/互补对比、彩色次要元素——调色已有自身体系，此处只补"色彩为构图服务"的目标 |
| review-cut 质检 | 跑下方「八条构图错误」清单 |

## 景别与运镜速查（数字镜头语言）

- **全景（建立镜头）**：交代"这是什么、元素怎么布局"。信息多、节奏慢。用于一节开头、上下文。
- **特写（punch-in 拉近）**：突出单一操作/单一元素，降复杂度。讲到具体动作时拉近，讲完拉回。
- **节奏来自变换**：全景与特写交替 = 蒙太奇节奏；一直全景=平、一直特写=憋。相邻 punch-in 间隔 ≥ 8s（与 G5b 一致）。
- **裁切要"接近"不要"切断"**：拉近裁掉的应是冗余留白，不能切断一个完整意义单元（半个按钮、半行关键句反而更乱）。
- **运镜方向**：与阅读方向一致（左→右、由小到大）= 加速、积极；逆向 = 减速、迟疑。拉近要平滑缓入缓出，抖动=技术缺陷。

## 视觉重心布局（破"居中呆板"）

- 主要元素**永远摆正中**会显得静止、呆板、乏味——这是新手最常见错误。
- 想要安静稳定 → 居中或对称；想要和谐 → 三分线/黄金分割点；想要张力动感 → 靠近边缘。
- 录屏天然偏居中，**靠 punch-in 重新构图**：把目标区域放到画面偏上/三分位，留白给信息呼吸。
- 一画面**只立一个主元素**；其余降为次要（补充、不抢戏）或删除（干扰留白）。

## 八条构图错误（质检清单 · review-cut 时逐条过）

录屏/出镜语境下逐条检查，命中即记入 `work/qa-report`：

- [ ] **画面抖动/不清** — 运镜不平滑、抽帧目标区不清晰；峰值帧主体必须清晰且在中心区（G5b 已检）。
- [ ] **死板居中** — 主元素长期钉在正中、四周大片无意义留白。该 punch-in 重构图或换景别。
- [ ] **主题重叠** — 前景主体与背景元素糊在一层、轮廓难辨（录屏：弹窗压住主操作区；出镜：人头与背景物重叠）。换机位/拉近分离。
- [ ] **线条混乱** — 画面里太多朝向不一的线/UI 元素，视线无处落。突出一条主导线指向角落，弱化杂线。
- [ ] **硬光阴影干扰**（出镜/实拍）— 脸上硬阴影盖住表情或眼睛；柔光/补光/换朝向。
- [ ] **背景混乱** — 背景出现彩色块/亮区/抢眼形状抢走主体注意力。换位置、虚化、或等干扰物移开。
- [ ] **元素过多/密集** — 一屏信息过载、无主次，观众要自己找重点。靠近主体、拆成多屏、"少即是多"。
- [ ] **倾斜地平线/水平线**（实拍）— 除非刻意求动感，水平线必须与画框平行。

## 关注度 vs 可信度（取舍原则）

形式上越花哨（怪角度、强运镜、夸张构图）→ 越吸睛但越"不真实"；越朴素标准 → 越可信但越易被忽略。
**教程/产品类要可信**：构图克制、标准焦段感、稳。**Hook/转场可适度花哨**博关注。按 brief 的调性定档。

## 进化闭环

本 skill 是**原理依据**；当某次构图判断被用户确认为好/被纠正，按管线惯例把它**提炼成一条规则补进 `pipeline/taste.md`**（构图/运镜节），让判断沉淀为可复用资产。本文件只承载"为什么"，taste.md 承载"本频道实际怎么做"。
